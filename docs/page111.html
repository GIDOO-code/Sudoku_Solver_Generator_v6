Sudoku Algorithm 6.0

 Puzzle Creation

GNPX has three ways to Create Sudoku puzzles:

 (1) Puzzle Creation

Manually create Sudoku puzzles in the 9x9 square.

  1. First, create a strong image in your mind of the Digit you want to enter.
  2. When you click on the position in the 9x9 square where you want to enter the digit, the digit you imagine will appear in the specified position.
  3. If the digit different from what you imagined appears, it is because your imagination power was weak.
    In that case, try to imagine it again more strongly and click. You can try this as many times as you like.
  4. To cancel the digit you have already entered, strongly think "disappear" and click on the digit again.


 (2) Puzzle auto-generation

GNPX-v6 generates Sudoku puzzles with varying board patterns.
In particular, steps 4 and 5 are useful methods for generating Sudoku puzzles.
In GNPX ver.6, steps 3 and 5 are executed in Parallel.

step-1
Set Pattern		 Specify the pattern's symmetric type (point symmetry, line symmetry)/asymmetric type and the number of pattern cells to automatically create a pattern.
The pattern controls the change with three parameters:
  1) Maximum number of puzzles in one pattern
  2) Number of Lattin-Square (maximum 3136)
step-2
LS Frame generation		 (LS: Latin Square)
Block 1 is fixed to 1 to 9. Select blocks 23 and block 47 (1/3136).
(There are 56 patterns in block 23 and 56 patterns in block 47, for a total of 3136 patterns.)>
step-3
LS generation [Parallel]		 Generate all LS squares from the LS frame. The number of LS that can be generated varies depending on the LS frame (2000-4000).
step 4
Puzzle candidate selection/td> 		Create a list of puzzle candidates by overlaying the pattern from step-1 on the LS from step-3.
By overlapping the patterns, the same puzzle can be created. Select only unique patterns from the puzzle candidate list.
A puzzle that has duplicates will have more than two patterns when the Sudoku algorithm is applied, which violates the rules of Sudoku puzzles.
This selection process eliminates approximately 50%.
step 5
Try&Error applied [Parallel]		 Solve the puzzle in step 4 using Try & Error and select the puzzle that has the unique solution.
(T&E patterns with multiple solutions are not Sudoku Puzzles and cannot be solved by applying Sudoku analysis algorithms.)
Depending on the pattern, it is quite likely that there is no unique solution.
The remaining number, if any, is between 0 and 10%, and these remaining pieces are guaranteed to be Sudoku Puzzles.
step 6
Algorithm Application		 Verify that the unique solution puzzle in step 5 can be solved using the Sudoku algorithm.
An unsolvable puzzle means that the analysis algorithm is weak.


 (3) Generate all puzzles of a specific pattern

GNPX generates all Sudoku puzzles for a given board pattern, which is relatively simple and therefore parallelizable.
Unfortunately, it doesn't take as long as you might think.
